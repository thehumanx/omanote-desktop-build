# What's new & changelogs

omanote ships updates regularly. The app lets you know when something new is available, and you can always browse the full history.

## How you find out about updates

When a new version ships, a **notification banner** appears at the bottom of the screen: *"New update · vX.X.X"* with a summary of what changed. Tap it to open the changelog modal, or dismiss it. If multiple updates have accumulated since you last checked, the banner shows *"+ N more updates"*.

The modal has three tabs:

- **Webapp** — updates to the main web app at omanote.com
- **Desktop** — releases of the native desktop app
- **Extension** — versions of the browser extension

Each tab shows version cards with the version number, date, summary line, and bullet-point changes. The latest version opens expanded; older versions are collapsible.

## Seeing the full history

A **"View all changelogs"** link in the modal takes you to `/updates`, a dedicated page that shows the complete version history alongside a roadmap of what's coming next.

## How updates are checked

The web app checks for new versions every 5 minutes in the background, and again whenever the tab regains focus. The desktop app checks every 30 minutes via its built-in updater. When a new version is found, the banner appears — tap "Refresh to update" to reload with the latest code.

For the browser extension, updates are handled by the Chrome Web Store and Firefox Add-ons. The modal shows a link to update the extension from the store when a new version is available there.

## Roadmap

The `/updates` page also includes a **Now / Next / Later** roadmap so you can see what's being worked on, what's coming soon, and what's planned further ahead. It's updated periodically and reflects the current priorities.
